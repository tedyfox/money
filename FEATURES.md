## Пользовательские сценарии

### Сценарий 1: Первый запуск

- При отсутствии токена в `localStorage` показывается **модалка ввода API-токена**
- Пользователь вводит токен → нажимает Enter или кнопку «Сохранить»
- Токен сохраняется в `localStorage` под ключом `api_token`
- Модалка закрывается, показывается форма расхода

### Сценарий 2: Добавление расхода

1. Поле суммы **в фокусе автоматически** при открытии страницы
2. Пользователь вводит сумму (десятичное число > 0)
3. Выбирает валюту — пилюли: **RSD** (по умолчанию), RUB, USD, EUR, GEL
4. Выбирает категорию из **16 вариантов** в двухколоночной сетке (список ниже)
5. Выбирает дату (по умолчанию — сегодня)
6. Опционально добавляет заметку (макс. 500 символов)
7. Нажимает «Сохранить»

**Состояния кнопки и формы:**
- `idle` — обычный вид
- `disabled` — кнопка неактивна (opacity 30%), если форма невалидна
- `loading` — текст «Сохраняю…»
- `success` — показывается: галочка, «Расход сохранён», сумма в RSD, кнопка «Добавить ещё»
- `error` — красное сообщение об ошибке, можно повторить попытку

После нажатия «Добавить ещё» — форма сбрасывается, возврат к состоянию `idle`.

### Сценарий 3: Смена токена

- Внизу главной страницы всегда видна кнопка **«Сменить токен»**
- При ответе сервера `401` — токен автоматически сбрасывается, показывается модалка

### Сценарий 4: Просмотр аналитики

- Отображается текущий месяц (название по-русски + год)
- Кнопки навигации: **← предыдущий / следующий →**
- Кнопка «вперёд» задизейблена для текущего и будущих месяцев
- Если нет записей за месяц → текст «**Нет записей за этот месяц**»

### Сценарий 5: Настройка бюджета

- Сумма месячного бюджета — **кликабельный инлайн-редактируемый элемент**
- Хранится в `localStorage` (`budget_monthly_rsd`), значение по умолчанию: **210 600 RSD**
- Под бюджетом отображается:
  - Остаток после фиксированных расходов
  - Дневной лимит (остаток ÷ дней в месяце)
  - Недельный лимит (дневной × 7)

### Сценарий 6: Управление фиксированными расходами

- Список фиксированных расходов с **инлайн-редактированием** каждого поля (название, сумма, валюта)
- У каждой строки — кнопка удаления **×**
- Внизу списка — форма добавления: поля «название» и «сумма», селектор валюты, кнопки **+** (сохранить) и **✕** (отменить)
- Хранится в `localStorage` (`budget_fixed_expenses`)

**Значения по умолчанию:**
| Название | Сумма | Валюта |
|---|---|---|
| Аренда | 800 | EUR |
| Коммуналка | 25 000 | RSD |
| Кредит | 21 560 | RUB |

## Блоки аналитики (все обязательны)

### По неделям
- Расходы разбиты на 7-дневные бакеты: 1–7, 8–14, 15–21, 22–конец месяца
- Если сумма за неделю превышает недельный лимит → **красная подсветка**

### По категориям
- Горизонтальные бары для каждой категории
- Ширина бара пропорциональна максимальной категории (100% = самая дорогая)
- Показывает: название категории и сумму в RSD

### Все записи
- Полный список расходов за месяц
- Для каждой записи: дата (формат ru-RU), категория, заметка (если есть), сумма в RSD
- Если валюта не RSD — дополнительно показывается оригинальная сумма с валютой
- Сортировка по дате — от новых к старым

## Обработка ошибок

| Ситуация | Поведение |
|---|---|
| Нет токена | Показать модалку токена |
| 401 от сервера | Сбросить токен, показать модалку |
| Сеть недоступна | «Нет соединения с сервером» |
| FX API недоступен | 502, сообщение «Не удалось получить курс валюты» |
| Валидационная ошибка | Поле-специфичное сообщение на русском |
| Ошибка сервера (5xx) | Показать сообщение об ошибке |
